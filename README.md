# 안심체크 — 이 문자 진짜예요?

부모님 세대를 노리는 **스미싱·보이스피싱 문자를 붙여넣기만 하면 AI가 확인해 주는 웹 서비스**입니다.

- **바로 써보기:** https://anshim-check.vercel.app
- 설치·가입·로그인 없이 링크만 열면 바로 사용
- 모바일 브라우저에서 **홈 화면에 추가**하면 앱처럼 바로 켤 수 있어요

> 2026 리부트 AI 활용대회 산출물. 국내 LLM **Upstage Solar**를 핵심 엔진으로 사용했습니다.

---

## 왜 만들었나

- 사기 문자·전화는 계속 늘고, 특히 **어르신이 가장 취약**합니다.
- 사기범은 **불안과 다급함**을 무기로 씁니다. 정작 위급한 순간엔 자녀에게 물어볼 생각조차 못 합니다.
- 기존 확인 수단은 **앱 설치·계좌·로그인**을 요구해서, 정작 필요한 어르신이 쓰기 어렵습니다.

그래서 **설치도 가입도 없이, 문자를 붙여넣기만 하면 되는** 가장 낮은 문턱을 목표로 했습니다.

## 무엇을 하나 (기능)

1. **사기 문자 확인** — 의심스러운 문자·카톡을 붙여넣으면, **국내 AI Solar**가 `위험 점수(0~100)` + `높음/중간/낮음` + `의심되는 이유` + `지금 할 일`을 쉬운 말로 알려 줍니다.
2. **예시 카드** — 아직 붙여넣을 문자가 없어도 건강보험·택배·검찰·가족사칭 예시를 눌러 바로 체험할 수 있습니다.
3. **위험한 부분 하이라이트** — 확인한 문자 안의 링크·전화번호·계좌형 숫자·위험 키워드를 노랗게 표시합니다.
4. **가족에게 보내기** — 결과 요약(점수·이유·원문 일부)을 공유 시트 또는 클립보드로 가족에게 전달합니다.
5. **읽어주기** — 결과 점수와 지금 할 일을 한국어 TTS로 낭독합니다. (지원 기기만)
6. **지우고 다시** — 입력·결과를 비우고 바로 다음 문자를 확인할 수 있습니다.
7. **홈 화면에 추가 (PWA)** — manifest + service worker로 오프라인 셸·standalone 실행을 지원합니다.
8. **통화 중 3원칙** — 실시간 전화는 붙여넣을 수 없으므로, "정부·은행은 전화로 계좌를 묻지 않는다 / 일단 끊고 대표번호로 / 헷갈리면 가족에게 캡처" 3원칙을 항상 화면에 고정 노출합니다.
9. **실종자 함께 찾기** — 경찰청 안전Dream OpenAPI 공개 정보(아동·어르신·발달장애 실종자)를 앨범형 보드로 보여 줍니다. 스와이프·자동 넘김·5분 갱신. (광고 슬롯 대신 공익 정보)
10. **공식 상담 안내** — 헷갈릴 때 **피싱안심SOS** · 상담전화 **1394** · 금감원 **1332** · 실종 **182**로 연결합니다.

## AI 활용 & 기술 스택

| 구분 | 사용 |
|---|---|
| **핵심 판단 AI (메인)** | **Upstage Solar Pro 3** (`solar-pro3`) — 문자 위험도 판정·근거·대처법 생성 |
| 프론트엔드 | HTML / CSS / JavaScript (프레임워크 없이 가볍게, 시니어 가독성 우선) |
| PWA | `manifest.webmanifest` + `sw.js` (정적 셸 cache-first, `/api/*` network) |
| 서버리스 | Vercel Functions (Node.js) — API 키를 서버에만 보관해 노출 방지 |
| 외부 데이터 | 경찰청 안전Dream 실종정보 OpenAPI |
| 개발 보조 | Claude Code / Codex / Grok (코딩·QA 보조 도구로만 사용) |

**Solar를 어떻게 썼나:** 단순 호출이 아니라, 실제 사기 유형을 다섯 가지(링크형·사칭협박형·기관콜백형·가족긴급송금형·링크미끼형)로 정리해 **한국어 판정 프롬프트를 직접 설계**했습니다. "입력에 없는 내용을 지어내지 말 것", "애매하면 낮음이 아니라 중간으로" 같은 안전 규칙을 넣어, 실제 사기를 안전하다고 오판하거나 평범한 가족 문자를 사기로 몰지 않도록 조정했습니다.

## 만든 과정 (시행착오)

- Solar 프롬프트 초기 버전은 **근거는 사기라면서 위험도는 '낮음'으로 판정**하는 모순, 그리고 **평범한 문자에 없는 위험요소를 지어내는** 문제가 있었습니다. 판정 규칙을 반복 수정해 잡았습니다.
- 실제 사기/정상 문자 표본으로 QA(어려운 사례 15건·페르소나 10건·점수제)를 돌려 정확도를 확인했습니다. (`_qa_scam_*.py`, `_qa_score.py`, 리포트 JSON 참고)
- 실종정보 API의 사진이 URL이 아니라 **base64 원문**으로 온다는 걸 QA 중 발견해, data URI로 처리하도록 고쳤습니다.
- 하단 영역을 **광고 대신 공익 실종 보드**로 두어, 확인 직후 시선이 머무는 자리를 공익 정보로 채웠습니다.

## 개인정보 · 윤리

- 입력한 문자 내용을 **저장하지 않습니다.** (서버는 API 키만 보관)
- 실종정보는 경찰청 공개 자료이며 화면에 **자료 출처: 경찰청**을 표기하고 무단 영리 이용을 금합니다.
- AI 판정은 **참고용**입니다. 확실하지 않으면 반드시 **피싱안심SOS·1394·1332** 또는 가족에게 확인하도록 안내합니다.
